# RELIABILITY.md — 안정성 가이드

## 에러 처리 원칙

- API 호출 실패 시 사용자에게 명확한 메시지 표시
- 진단 API 실패 시 1회 자동 재시도 후 error 상태
- localStorage 접근 실패 (용량 초과 등) 조용히 무시
- 이미지 파싱 실패 시 500 대신 의미있는 에러 메시지 반환

## 폴백 체계

```
FAQ 매칭 실패 → 캐시 확인 → API 호출 → 에러 메시지
진단 API 실패 → 1회 재시도 → error 상태 + 다시 시도 버튼
사진 분석 실패 → 에러 메시지 (파일 형식/크기 안내 포함)
```

## 알려진 제약

- `ANTHROPIC_API_KEY` 미설정 시 모든 AI 기능 동작 불가
- localStorage 비활성화 환경에서 캐시 미동작 (정상 폴백)
- 이미지 5MB 초과 시 업로드 거부 (클라이언트 사전 검사)
